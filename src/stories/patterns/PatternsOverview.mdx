# Patterns

Common UI patterns composed from our components. These represent recurring solutions to design problems.

## Available Patterns

| Pattern | Description |
|---------|-------------|
| **[Dashboards](/docs/patterns-dashboards--docs)** | Stats grids, KPI cards, activity feeds, and pipeline status |
| **[Forms](/docs/patterns-forms--docs)** | Simple forms, multi-step wizards, inline editing, and search filters |
| **[Lists](/docs/patterns-lists--docs)** | Data tables, card grids, expandable items, and empty states |

## When to Use Patterns

Patterns are complete UI solutions that combine multiple components:

- **Dashboards** — For overview pages and status displays
- **Forms** — For data entry and configuration
- **Lists** — For displaying and managing collections of items

Each pattern includes common variations and states (loading, empty, error).
